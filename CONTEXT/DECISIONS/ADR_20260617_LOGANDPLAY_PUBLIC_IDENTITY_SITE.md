# ADR 2026-06-17 — LOGANDPLAY public identity site

## Status

Accepted.

## Context

The Linux preproduction server is now hardened enough to prepare the next public-facing identity layer, but OPUS, MAESTRO and KB are not publicly exposed yet.

The user wants `logandplay.org` to stop being an Apache default page and become a professional public entry point generated by OPUS.

OPUS remains the framework core. LOGANDPLAY becomes a separate workspace project/site identity layer.

## Decision

Create a dedicated LOGANDPLAY project in MAESTRO_WORKSPACE.

The future `logandplay.org` page will be the public identity card for the ecosystem:

```text
LOGANDPLAY
  official public identity and entry point

OPUS
  framework, libre d'utilisation personnelle / non commerciale according to OPUS license intent
  status shown as PROCHAINEMENT until exposed

MAESTRO
  music assistant / REAPER workflow project
  status shown as PROCHAINEMENT until exposed

KB / Maestro Knowledge Engine
  private musical knowledge and worker engine
  status shown as PROCHAINEMENT until an explicit public surface exists
```

The page must be generated by OPUS but the content must not be embedded in the OPUS framework core.

## Public exposure rule

Until Cloudflare Tunnel and Cloudflare Access are explicitly configured, the public page remains a preproduction target.

No Freebox port opening is authorized by default.

## Security rule

The public page must not expose:

```text
server paths
LAN hostnames
Webmin links
admin URLs
preproduction-only routes
internal diagnostics
private service topology
```

## Consequences

A future implementation should add an OPUS-compatible LOGANDPLAY site/package or site layer rather than hardcoding Log&Play business content in OPUS core classes.

The initial visual state of project links is `PROCHAINEMENT`, because OPUS, MAESTRO and KB are not public yet.
