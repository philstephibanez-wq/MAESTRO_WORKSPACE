## Binding OWASYS product contract

OWASYS is an OPUS deliverable for OPUS users.

- portable on supported Windows and Linux OPUS environments;
- no hard dependency on `H:/OPUS`, UwAmp, one HP computer, or one production server;
- installation root resolved at runtime;
- environment separation driven by `OPUS_ENV`, not operating system or machine identity;
- OPUS architecture remains state-first and has no generated `src`, `public`, or `resources` roots;
- OWASYS itself does not boot the generated application profiler.

Distribution contract:

- `sites/owasys/config/distribution.json`
- contract: `OWASYS_DISTRIBUTION_V1`

- production returns no profiler instance even with `?profiler=1`;
- current implementation is functional but must not be described as complete collector coverage.

## Source + Git core state

The first secure Source + Git core is implemented and smoke-validated.

### RepositoryInspector

File: `Opus/Owasys/RepositoryInspector.php`

- detects the Git repository associated with an application;
- exposes branch, HEAD, clean/dirty state, recent history and diff;
- read-only only at this stage;
- no free-form command execution;
- refuses repository/path resolution outside the OPUS root.

### ApplicationFileEditor

File: `Opus/Owasys/ApplicationFileEditor.php`

- lists and reads editable application files;
- editable roots limited to `config/`, `application/` and `www/asset/`;
- refuses absolute paths, traversal, `.git`, secrets and authentication stores;
- supports diff preview without disk mutation;
- requires the current SHA-256 fingerprint before write;
- validates PHP syntax and JSON before write;
- performs atomic replacement;
- enforces file-size limits.

Validated marker:

- `OWASYS_SOURCE_GIT_CORE_SMOKE_OK`
